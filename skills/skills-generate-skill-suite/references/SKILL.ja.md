# スキルスイートを生成する

複数の関連する単一ワークフロースキルをルータースキル、一貫した命名、クロスリファレンスを含む協調されたスイートとして生成するワークフロー。レガシーの「マルチパターンスキル」の代替手段。

## When to Use This Skill

このスキルを使用する場面：
- ドメインをファミリーを構成する複数の単一ワークフロースキルに分解する時
- ルータースキルと構成ワークフロースキルをまとめて生成する時
- スキルファミリー全体で一貫した命名、タグ、クロスリファレンスを確保する時
- レガシーのマルチパターンスキルを新しいワークフロースキルのスイートに移行する時
- 最初から3つ以上の協調されたスキルが必要な新ドメインを計画する時

---

## Core Principles

1. **スイート＝協調されたファミリー** — スイートは一貫した命名、共有コンテキストプレフィックス、明示的クロスリファレンスを持つスキルの集合（基礎と型）
2. **ルーターがエントリポイント** — すべてのスイートにはユーザーが正しいワークフローを見つけるためのルータースキルがある（ニュートラル）
3. **独立だが関連** — スイート内の各スキルは自己完結的; スイートは依存ではなく発見性を追加（成長の複利）
4. **まとめて生成、個別に埋める** — 一貫性のためにすべてのスケルトンを一度に生成; 各々を個別に埋める（継続は力）

## ワークフロー：スキルスイートを生成する

### ステップ1 — スイートを定義

スキルファミリーを計画：コンテキストプレフィックス、ルーター、ワークフロースキル一覧。

**Why**: 計画が最初です。命名の衝突、ワークフローの抜け、不整合なクロスリファレンスを避けるため。基礎と型 — 設計してから実装。

### ステップ2 — 全スケルトンを生成

`skills-generate-skill-template`を使用してスイート内の各スキルのスケルトンを生成。

### ステップ3 — ルータースキルを生成

すべてのスイートメンバーをリンクするルータースキルを作成（または変換）。

**Why**: ルーターはスイートの「目次」。ニュートラルな視点 — 誰もが適切なワークフローを見つけられるように。

### ステップ4 — クロスリファレンスを追加

各スキルの"Related Skills"セクションが他のスイートメンバーを参照するように。

### ステップ5 — 一貫性を検証

全スイートメンバーが共有プレフィックス、一貫したタグ、クロスリファレンス、同じ構造テンプレート、`author: RyoMurakami1983`を持つことを確認。

### ステップ6 — 各スキルを埋めて検証

`skills-author-skill`で各スケルトンを埋め、`skills-validate-skill`で検証。

## Good Practices

### 1. 生成前にスイートを計画

**What**: ファイル生成前にスイート計画（コンテキスト、スキル、ルーター）を書く。
**Why**: 計画が命名の衝突と不整合を防ぐ。
**Values**: 基礎と型

### 2. スイートは小さく保つ（3〜7スキル）

**What**: スイートは3〜7ワークフロースキル＋ルーター。
**Why**: 少なすぎ（1〜2）はスイートの正当性がない; 多すぎ（8+）はナビゲーション困難。
**Values**: ニュートラル
